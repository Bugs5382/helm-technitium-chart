🚀 Technitium DNS Server has been successfully deployed!

1. 🔐 Get your Admin Password:
   Run the following command to retrieve the password for the 'admin' user:

   kubectl get secret --namespace {{ .Release.Namespace }} {{ include "technitium.fullname" . }}-admin -o jsonpath="{.data.password}" | base64 --decode; echo

2. 🌐 Access the Web UI:
{{- if .Values.ingress.enabled }}
   Ingress is enabled. Access the UI via the configured host(s):
{{- range .Values.ingress.hosts }}
     - http{{ if $.Values.ingress.tls }}s{{ end }}://{{ .host }}
{{- end }}
{{- else if eq (.Values.service.web.type | default "ClusterIP") "LoadBalancer" }}
   The web service is type LoadBalancer. Get the external IP:
   kubectl get svc --namespace {{ .Release.Namespace }} {{ include "technitium.fullname" . }}-web
{{- else }}
   Port-forward to access the UI at http://127.0.0.1:{{ .Values.ports.webHttp | default 5380 }}:
   kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "technitium.fullname" . }}-web {{ .Values.ports.webHttp | default 5380 }}:{{ .Values.ports.webHttp | default 5380 }}
{{- end }}

3. 📡 DNS Endpoint:
{{- if eq (.Values.service.dns.type | default "LoadBalancer") "LoadBalancer" }}
   The DNS service is type LoadBalancer. Get its external IP:
   kubectl get svc --namespace {{ .Release.Namespace }} {{ include "technitium.fullname" . }}-dns
{{- else }}
   The DNS service is type {{ .Values.service.dns.type }}. Inspect it with:
   kubectl get svc --namespace {{ .Release.Namespace }} {{ include "technitium.fullname" . }}-dns
{{- end }}
